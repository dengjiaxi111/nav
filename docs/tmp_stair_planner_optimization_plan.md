# 台阶段 Planner 优化方案（临时文档）

> 目的：针对“台阶前后路径应尽量垂直台阶法向”但当前 B-spline 优化效果不佳的问题，给出可落地的最优方案。
> 当前状态：`Draft v0`（仅讨论，不改代码）

## 1. 问题本质

当前实现的台阶对齐项存在三类天然/实现限制：
1. 约束施加在控制点切向（control polygon）而非最终曲线切向，控制点不在曲线上，约束会衰减。
2. 台阶命中依赖“控制点落在 stair_normal_valid 栅格”，命中稀疏时约束直接失效。
3. 代价函数用 `||tangent_perp||^2`，同时耦合角度和切向长度，不是纯姿态对齐。

结论：仅靠现有控制点级软约束，难以稳定实现“台阶前后固定范围内接近垂直”的工程效果。

## 2. 目标定义（验收口径）

对每次上台阶路径，定义台阶局部坐标系：
1. `n`：台阶法向（低侧->高侧）
2. `t`：台阶切向（边沿方向）

在“台阶作用窗”内，目标是：
1. 航向目标：路径切向 `T(s)` 与 `n` 对齐（`|T·n|` 接近 1）。
2. 横向目标：路径在 `t` 方向偏移受限（避免沿边滑行）。
3. 可行性目标：不过障碍、曲率与控制可跟踪性不劣化。

## 3. 推荐的最优方案（最终形态）

采用“三层约束”而非单一代价：

## 3.1 台阶语义对象化（Stair Primitive）

从 `stair_mask` 生成稳定对象：
1. `stair_id`
2. `center`
3. `normal n`
4. `tangent t`
5. `half_length`
6. `crossing_band`（作用窗）

目的：Planner 不再靠“偶然命中某个法向栅格”，而是对明确台阶对象施加约束。

## 3.2 参考路径模板化（几何先验）

当检测到“路径将跨越某个 stair_id 上行方向”时，先构造几何模板：
1. 入口锚点 `P_pre = center - n * pre_dist`
2. 中心锚点 `P_mid = center`
3. 出口锚点 `P_post = center + n * post_dist`
4. 锚点在 `t` 方向投影约束到中垂线附近（`|d_t| <= corridor_half_width`）

这一步是“先把拓扑和几何骨架摆正”，避免优化器在错误形态上微调。

## 3.3 曲线级优化（核心）

将台阶对齐从“控制点级”升级为“采样曲线级”：
1. 在作用窗按弧长采样曲线点 `x(s_i)` 与切向 `T(s_i)`。
2. 角度代价改为纯方向项：
- `J_heading = Σ w_i * (1 - (T(s_i)·n)^2)`
3. 横向走廊代价：
- `J_lateral = Σ w_i * huber(d_t(s_i) / corridor_half_width)`
4. 锚点代价（软硬结合）：
- `J_anchor = ||x(s_pre)-P_pre||^2 + ||x(s_mid)-P_mid||^2 + ||x(s_post)-P_post||^2`
5. 与平滑/碰撞联合优化：
- `J_total = J_smooth + J_collision + J_heading + J_lateral + J_anchor`

说明：
1. 该形式直接约束最终可执行路径方向，不再间接依赖控制点几何。
2. `(T·n)^2` 同时兼容上下台阶方向（符号问题可通过上行/下行分支加权处理）。

## 4. 为什么这比当前方法稳定

1. 命中稳定：对象化台阶+作用窗，不再依赖控制点落点碰巧命中法向栅格。
2. 约束直达目标：直接优化曲线切向，不受“控制点不在曲线上”影响。
3. 形态先验更强：锚点先把过台阶几何骨架固定，优化不易陷入局部最优。
4. 可解释性强：每个失效都能映射到“检测、模板、曲线约束、碰撞冲突”中的某一层。

## 5. 分阶段落地步骤（建议顺序）

## 阶段A：诊断增强（低风险）

1. 增加调试输出：
- 每条路径命中的 `stair_id`、作用窗范围、采样点数量。
- 台阶角度误差统计（均值/最大值）。
2. 增加 RViz 标记：
- `n/t` 方向、`P_pre/P_mid/P_post`、作用窗边界。

验收：能一眼看出“为什么没垂直”。

## 阶段B：快速收益改造

1. 保留现有 B-spline 框架。
2. 将台阶对齐项从控制点级改为曲线采样级（先用数值梯度近似可接受）。
3. 保留现有碰撞/平滑项，避免全盘重写。

验收：台阶段平均角度误差明显下降。

## 阶段C：模板锚点与走廊

1. 引入 `P_pre/P_mid/P_post` 锚点约束。
2. 引入 `d_t` 走廊约束，抑制沿台阶边滑行。
3. 上/下台阶参数分离。

验收：台阶前后路径形态稳定，重规划之间一致性高。

## 阶段D：鲁棒性与实时性

1. 将数值梯度替换为解析梯度或半解析梯度。
2. 增加信赖域/步长保护，避免强约束造成发散。
3. 加入冲突仲裁：当碰撞项与台阶项冲突时按优先级降权。

验收：50Hz 控制下规划延迟可接受，失败率可控。

## 6. 参数建议（初值）

1. `stair_pre_dist_m = 1.0`
2. `stair_post_dist_m = 0.5`
3. `stair_corridor_half_width_m = 0.25 ~ 0.35`
4. `stair_heading_weight = 30 ~ 80`
5. `stair_anchor_weight = 20 ~ 60`
6. `stair_lateral_weight = 10 ~ 40`
7. `stair_sample_ds_m = 0.05 ~ 0.1`

## 7. 定量验收指标

1. 角度指标：作用窗内 `|acos(T·n)|` 的 P95 <= 8°。
2. 位置指标：作用窗内 `|d_t|` 的 P95 <= `corridor_half_width`。
3. 安全指标：路径最小 ESDF 距离不低于当前基线。
4. 计算指标：优化耗时 P95 不超过当前实现 1.5 倍。
5. 业务指标：上台阶一次成功率显著提升（实测场地统计）。

## 8. 风险与回滚策略

1. 风险：强几何约束导致局部不可行。
- 处理：约束软化+权重退火+冲突时自动降权。
2. 风险：计算量上升。
- 处理：仅在命中台阶对象时启用曲线级约束。
3. 风险：误检台阶对象导致异常路径。
- 处理：对象置信度门槛+无对象时退回普通优化。

回滚策略：
1. 任一阶段均保留 `planner.stair_segment_align_mode` 开关。
2. 可随时回退到当前旧实现（control-point stair align）。

## 9. 已确认决策（2026-04-07）

1. 不引入“台阶作用窗内短距离降速绕障”策略：
- 赛场台阶宽度仅较机器人宽约 20cm，窗口内若有障碍通常直接不可规划，不需额外绕障策略。
2. 台阶窗口内遇到动态障碍时：
- 优先避障（脱离窗口/重规划），不强行保持垂直提交。
3. 上/下台阶角度阈值：
- 采用不同阈值；
- 上台阶阈值应比下台阶更严格。

## 10. 默认参数（已设）

1. `stair_up_heading_err_max_deg = 8.0`
2. `stair_down_heading_err_max_deg = 15.0`
3. `stair_window_escape_timeout_s = 0.5`

说明：
1. 上台阶阈值更严格，下台阶更宽松。
2. `stair_window_escape_timeout_s` 归属 `terrain_controller / stair_handler`，
   用于“进入台阶作用窗后遇到动态障碍时”的脱离窗口超时控制。
