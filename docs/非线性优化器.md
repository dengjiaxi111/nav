# ROS2 + NMPC (CasADi + acados) 项目环境配置指南

本项目将使用 **CasADi 构建系统动力学模型**，并通过 **acados 生成实时 NMPC 控制代码**，最终在 ROS2 Humble + C++ Controller 节点中运行，目标 50Hz+ 控制频率。

本文件旨在帮助你从零搭建环境，并说明如何将建模脚本与 ROS2 workspace 整合。

---

## 1. 依赖环境

推荐环境：

- Ubuntu 22.04
- ROS2 Humble
- GCC >= 9.4
- Python 3.10 / 3.12（本仓已适配 3.12）
- CMake >= 3.16

> acados 会通过 Python 依赖生成求解器，然后编译成高性能 C 代码供 C++ 直接使用。

---

## 2. 安装 CasADi

CasADi 可通过 pip 安装（包含自动微分核心功能）。

> 注意：`acados_template` 新版本会提示“完整功能建议 CasADi >= 3.7”。
> 本仓当前导出脚本在 `casadi==3.6.7` 下可工作，但若后续使用更高级功能，建议升级 CasADi 并做回归测试。

```
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip setuptools wheel
python -m pip install casadi==3.6.7
```

验证是否可用：

```
python3 - <<EOF
import casadi as ca
x = ca.SX.sym("x")
print("CasADi OK:", x)
EOF
```

---

## 3. 安装 acados（推荐源码编译，且与 `acados_template` 同源）

### 克隆仓库

```
mkdir -p ~/dependency
git clone https://github.com/acados/acados.git --recursive ~/dependency/acados
cd ~/dependency/acados
```

### 安装依赖

```
sudo apt install -y \
    cmake build-essential \
    libblas-dev liblapack-dev \
    libqpOASES-dev libeigen3-dev \
    python3-matplotlib python3-numpy python3-scipy python3-pip
python -m pip install -r requirements.txt
python -m pip install -e ~/dependency/acados/interfaces/acados_template
```

### 编译

```
mkdir -p build && cd build
cmake .. \
    -DACADOS_WITH_QPOASES=ON \
    -DACADOS_WITH_OSQP=ON \
    -DACADOS_INSTALL_DIR=$HOME/dependency/acados
make -j$(nproc)
make install
```

完成后（本仓推荐）目录结构：

```
~/dependency/acados/lib/libacados.so
~/dependency/acados/include/acados
~/dependency/acados/cmake/acadosConfig.cmake
```

验证：

```
python3 - <<EOF
import acados_template
print("acados_template OK:", acados_template.__file__)
EOF
```

若有输出即成功。

> 补充：`acados_template --help` 不是可靠验证方式。`acados_template` 主要作为 Python 包使用，不一定提供同名命令行程序。

### 3.1 安装 `t_renderer`（必须）

`export_ocp.py` 渲染模板时需要 `t_renderer`，若缺失会在导出阶段失败。

```bash
mkdir -p ~/dependency/acados/bin
curl -L --fail -o ~/dependency/acados/bin/t_renderer \
    https://github.com/acados/tera_renderer/releases/download/v0.2.0/t_renderer-v0.2.0-linux-amd64
chmod +x ~/dependency/acados/bin/t_renderer
```

## 4. 项目目录建议结构

```
ros2_ws/
├── src/
│   ├── wheelleg_control/          ◀ 你的controller包 (C++)
│   ├── nmpc_solver/               ◀ acados生成代码目录
│   ├── model_ocp/                 ◀ CasADi 建模 + OCP导出脚本
│   │   ├── model.py
│   │   ├── export_ocp.py          ◀ 运行后生成 solver
│   │   └── requirements.txt
│   └── wheelleg_msgs/
└── install/
└── build/
```

作用说明：

| 文件夹 | 内容 |
|---|---|
| model_ocp/ | 你用 CasADi 定义动力学模型 + OCP |
| nmpc_solver/ | acados 自动生成的求解器源码，C/C++ 可直接调用 |
| wheelleg_control/ | ROS2 Controller，加载 solver，订阅 odom 输出 cmd_vel 或 acc |

## 6. 在 C++ ROS2 Controller 中使用 solver

在 `CMakeLists.txt` 中链接：

```
target_link_libraries(wheelleg_control
    acados
    hpipm
    blasfeo
    # 本仓导出的 solver 动态库
    ${CMAKE_SOURCE_DIR}/../nmpc_solver/libacados_ocp_solver_wheelleg_nmpc.so
)
```

Controller 伪代码示例：

```cpp
#include "acados_solver_wheelleg.h"   // export_ocp.py 自动生成

acados_ocp_capsule *capsule = wheelleg_acados_create();

while(running){
    set_state(...)
    set_reference(...)
    wheelleg_acados_solve();
    read_control_output(...)
}
```

## 7. 已踩坑与对策（精简版）

1. **Python 3.12 + 旧 pin 冲突**  
    现象：安装 `numpy==1.24.3` 构建失败（`pkgutil.ImpImporter` 报错）。  
    对策：在 `requirements.txt` 用 `python_version` 条件区分 3.12 与 <=3.11 版本。

2. **`acados_template` 与 `libacados.so` 版本不匹配**  
    现象：`undefined symbol: ocp_nlp_dump_last_qp_to_json`。  
    对策：必须让 `acados_template` 与 `libacados.so` 来自同一份 `~/dependency/acados` 源码构建。

3. **缺少 `t_renderer` 导致导出中断**  
    现象：导出提示找不到模板渲染器。  
    对策：下载 `t_renderer` 到 `~/dependency/acados/bin` 并 `chmod +x`。

4. **`ACADOS_SOURCE_DIR` / `LD_LIBRARY_PATH` 未设置**  
    现象：导出时找不到 `libacados.so` 或头文件。  
    对策：每次导出前显式设置这两个环境变量。

5. **acados API 弃用提示（非阻塞）**  
    - `AcadosOcp.dims.N` 迁移到 `AcadosOcp.solver_options.N_horizon`。  
    - `ocp.code_export_directory` 迁移到 `ocp.code_gen_opts.code_export_directory`。  
    建议在后续脚本清理中统一替换，减少 warning 噪声。
